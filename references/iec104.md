# IEC 60870-5-104 协议参考

## 帧格式

IEC 104 基于 TCP/IP，端口 2404。每个 APDU 包含 APCI（应用规约控制信息）和 ASDU（应用服务数据单元）。

```
┌───────────────────────────────────────────────────────────────┐
│                            APDU                               │
├───────────────────────────────────┬───────────────────────────┤
│            APCI                   │          ASDU             │
│         (6 bytes)                 │       (可变长度)          │
└───────────────────────────────────┴───────────────────────────┘
```

## APCI 格式

```
┌──────────┬──────────┬─────────────────────────────────────────┐
│ 启动字节 │  长度    │              控制域                      │
│  1 byte  │  1 byte  │              4 bytes                     │
│   0x68   │   L      │    (格式取决于帧类型)                    │
└──────────┴──────────┴─────────────────────────────────────────┘
```

### 帧类型与控制域

| 帧类型 | 控制域格式 | 说明 |
|--------|------------|------|
| I帧 | [SendSeqL] [SendSeqH] [RecvSeqL] [RecvSeqH] | 信息传输 |
| S帧 | [01] [00] [RecvSeqL] [RecvSeqH] | 确认帧 |
| U帧 | [控制字节] [00] [00] [00] | 无编号控制帧 |

### U帧类型

| U帧类型 | 控制字节 | 说明 |
|---------|----------|------|
| STARTDT act | 0x07 | 启动数据传输激活 |
| STARTDT con | 0x0B | 启动数据传输确认 |
| STOPDT act | 0x13 | 停止数据传输激活 |
| STOPDT con | 0x23 | 停止数据传输确认 |
| TESTFR act | 0x43 | 测试帧激活 |
| TESTFR con | 0x83 | 测试帧确认 |

## ASDU 格式

```
┌──────────────┬──────────────┬──────────────┬─────────────────────┐
│   数据单元    │   可变结构    │   传输原因    │      公共地址        │
│   标识符      │   限定词      │               │                     │
│   (SIQ/TIQ)   │   (VSQ)      │   (COT)      │   (Common Addr)     │
│   1 byte      │   1 byte     │   2 bytes    │   2 bytes           │
├──────────────┴──────────────┴──────────────┴─────────────────────┤
│                        信息体                                    │
│              (Information Objects)                               │
│                      可变长度                                    │
└──────────────────────────────────────────────────────────────────┘
```

### 数据单元标识符 (TypeID)

| TypeID | 名称 | 说明 |
|--------|------|------|
| 1 | M_SP_NA_1 | 单点信息 |
| 3 | M_DP_NA_1 | 双点信息 |
| 9 | M_ME_NA_1 | 测量值，标度化值 |
| 11 | M_ME_NB_1 | 测量值，浮点数 |
| 13 | M_ME_NC_1 | 测量值，IEEE浮点 |
| 30 | M_SP_TB_1 | 带时标单点信息 |
| 35 | M_ME_TF_1 | 带时标测量值，浮点数 |
| 45 | C_SC_NA_1 | 单点命令 |
| 100 | C_IC_NA_1 | 总召唤命令 |

### 传输原因 (COT)

| COT | 名称 | 说明 |
|-----|------|------|
| 1 | 周期/循环 | 周期传输 |
| 3 | 突发 | 突发传输 |
| 5 | 请求 | 响应请求 |
| 6 | 激活 | 激活命令 |
| 7 | 激活确认 | 激活确认 |
| 20 | 响应站召唤 | 总召唤响应 |
| 44 | 未知的类型标识 | 错误响应 |

## 序列号机制

- **发送序列号 (Send Seq)**: I帧发送时递增，用于流量控制
- **接收序列号 (Recv Seq)**: 确认已收到的I帧
- **序列号范围**: 0-32767，循环使用
- **确认机制**: S帧或I帧携带接收序列号确认

### 序列号验证

```python
def check_sequence(expected, actual, window=8):
    """检查序列号连续性"""
    diff = (actual - expected) & 0x7FFF
    if diff == 0:
        return "序列号匹配"
    elif diff <= window:
        return f"序列号超前{diff}，可能丢包"
    else:
        return "序列号异常"
```

## 解析示例

### STARTDT 激活帧

```
输入: 68 04 07 00 00 00

解析:
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│ 启动字节 │  长度    │ 控制域1  │ 控制域2  │ 控制域3  │ 控制域4  │
│   68     │   04     │   07     │   00     │   00     │   00     │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘

- 启动字节: 0x68
- 长度: 4 (APCI长度，不含启动字节和长度字段)
- 帧类型: U帧
- U帧类型: STARTDT act (启动数据传输激活)
```

### I帧 - 单点信息

```
输入: 68 0E 02 00 01 00 01 01 03 00 01 00 01 00 01

解析:
APCI:
- 启动字节: 0x68
- 长度: 14
- 发送序列号: 1 (低字节02/2右移1位)
- 接收序列号: 0 (低字节01右移1位)

ASDU:
- TypeID: 0x01 = M_SP_NA_1 (单点信息)
- VSQ: 0x01 = 1个信息对象
- COT: 0x03 0x00 = 突发
- 公共地址: 0x01 0x00 = 1
- 信息对象地址: 0x01 0x00 0x00 = 1
- SIQ: 0x01 = 合(ON)
```

### S帧确认

```
输入: 68 04 01 00 02 00

解析:
- 启动字节: 0x68
- 长度: 4
- 帧类型: S帧 (控制域首字节低2位=01)
- 接收序列号: 1 (低字节02右移1位)
- 含义: 确认已收到序列号为1的I帧
```

## 常见问题

1. **序列号不连续**: 可能丢包，检查网络质量
2. **超时无确认**: 检查对方是否响应，调整超时参数
3. **TESTFR无响应**: 链路可能断开，检查TCP连接
4. **总召唤无响应**: 检查公共地址配置，确认设备支持该功能

## 时间戳格式 (CP56Time2a)

```
┌─────────────────────────────────────────────────────────────────┐
│  毫秒(低)  │  毫秒(高)  │ 分钟IV  │ 时SU   │ 日  │ 周  │ 月  │ 年  │
│  1 byte    │  1 byte    │  1 byte │ 1 byte │1byte│1byte│1byte│1byte│
└─────────────────────────────────────────────────────────────────┘

- 毫秒: 0-59999
- 分钟: 0-59 (bit7=IV, 有效性)
- 小时: 0-23 (bit7=SU, 夏令时)
- 日: 1-31
- 周: 1-7
- 月: 1-12
- 年: 0-99 (相对于2000年)
```
