# MQTT 协议参考

## 概述

MQTT (Message Queuing Telemetry Transport) 是一种轻量级的发布/订阅消息传输协议，广泛应用于物联网和工业自动化领域。

## 固定头部格式

```
┌──────────────────────────────────────────────────────────────┐
│                        固定头部                               │
├──────────┬──────────┬──────────┬──────────┬──────────────────┤
│  类型    │ DUP标志  │  QoS级别 │  RETAIN  │    剩余长度      │
│ 4 bits   │  1 bit   │  2 bits  │  1 bit   │   1-4 bytes       │
└──────────┴──────────┴──────────┴──────────┴──────────────────┘
```

### 控制报文类型

| 类型值 | 名称 | 方向 | 说明 |
|--------|------|------|------|
| 1 | CONNECT | Client→Broker | 客户端请求连接 |
| 2 | CONNACK | Broker→Client | 连接确认 |
| 3 | PUBLISH | 双向 | 发布消息 |
| 4 | PUBACK | 双向 | QoS 1发布确认 |
| 5 | PUBREC | 双向 | QoS 2发布接收 |
| 6 | PUBREL | 双向 | QoS 2发布释放 |
| 7 | PUBCOMP | 双向 | QoS 2发布完成 |
| 8 | SUBSCRIBE | Client→Broker | 订阅请求 |
| 9 | SUBACK | Broker→Client | 订阅确认 |
| 10 | UNSUBSCRIBE | Client→Broker | 取消订阅 |
| 11 | UNSUBACK | Broker→Client | 取消订阅确认 |
| 12 | PINGREQ | Client→Broker | 心跳请求 |
| 13 | PINGRESP | Broker→Client | 心跳响应 |
| 14 | DISCONNECT | Client→Broker | 断开连接 |
| 15 | AUTH | 双向 | 认证交换(MQTT 5.0) |

### 标志位说明

| 标志 | 位置 | 说明 |
|------|------|------|
| DUP | Bit 3 | 重复分发标志，表示这是重传的消息 |
| QoS | Bit 2-1 | 服务质量等级：0, 1, 2 |
| RETAIN | Bit 0 | 保留标志，表示消息需要保留 |

### 剩余长度编码

使用可变长度编码（1-4字节）：

| 值范围 | 字节数 |
|--------|--------|
| 0-127 | 1 |
| 128-16383 | 2 |
| 16384-2097151 | 3 |
| 2097152-268435455 | 4 |

编码算法：
```
每个字节的低7位为数据位，最高位表示是否还有后续字节
例如：321 = 0xC1 0x02 (321 = 65 + 2*128)
```

## CONNECT 报文

```
┌──────────────┬──────────────┬─────────────────────────────────┐
│   固定头部   │   协议名     │        协议级别                 │
│   10 ...     │  00 04 M Q T T  │         04 (MQTT 3.1.1)       │
└──────────────┴──────────────┴─────────────────────────────────┘
┌──────────────┬──────────────┬──────────────┬──────────────────┐
│   连接标志   │  保活时间   │   客户端ID   │     有效载荷     │
│   1 byte     │   2 bytes   │   可变长度   │     可变长度      │
└──────────────┴──────────────┴──────────────┴──────────────────┘
```

### 连接标志位

| 位 | 名称 | 说明 |
|----|------|------|
| 7 | User Name Flag | 是否包含用户名 |
| 6 | Password Flag | 是否包含密码 |
| 5 | Will Retain | 遗嘱消息保留标志 |
| 4-3 | Will QoS | 遗嘱消息QoS等级 |
| 2 | Will Flag | 是否包含遗嘱消息 |
| 1 | Clean Session | 清除会话标志 |
| 0 | Reserved | 保留位，必须为0 |

### 有效载荷顺序

1. 客户端ID (必须)
2. 遗嘱主题 (如果Will Flag=1)
3. 遗嘱消息 (如果Will Flag=1)
4. 用户名 (如果User Name Flag=1)
5. 密码 (如果Password Flag=1)

## PUBLISH 报文

```
┌──────────────┬──────────────────────────────────────────────────┐
│   固定头部   │            可变头部 + 有效载荷                    │
├──────────────┼──────────────┬──────────────┬──────────────────┤
│  3x DUP QoS  │   主题       │   报文标识   │      消息体       │
│  RETAIN ...  │   可变长度   │  (QoS>0时)  │     可变长度      │
└──────────────┴──────────────┴──────────────┴──────────────────┘
```

### QoS级别

| QoS | 说明 | 消息流 |
|-----|------|--------|
| 0 | 最多一次 | Client → Broker (无确认) |
| 1 | 至少一次 | Client → Broker → PUBACK → Client |
| 2 | 恰好一次 | Client → Broker → PUBREC → Client → PUBREL → Broker → PUBCOMP → Client |

## SUBSCRIBE 报文

```
┌──────────────┬──────────────┬──────────────────────────────────┐
│   固定头部   │   报文标识   │         有效载荷                 │
│   82 ...     │   2 bytes    │   主题过滤器 + QoS (可重复)      │
└──────────────┴──────────────┴──────────────────────────────────┘
```

### 主题过滤器格式

```
主题级别分隔符: /
单级通配符: +
多级通配符: #

示例:
  sensor/temperature    - 精确匹配
  sensor/+              - 匹配 sensor/temperature, sensor/humidity
  sensor/#              - 匹配 sensor 及其所有子主题
```

## 解析示例

### CONNECT 请求

```
输入: 10 1D 00 04 4D 51 54 54 04 C2 00 3C 00 0C 4D 51 54 54 5F 43 6C 69 65 6E 74

解析:
固定头部:
  类型: 0x1 = CONNECT
  剩余长度: 0x1D = 29字节

可变头部:
  协议名长度: 0x0004 = 4
  协议名: "MQTT"
  协议级别: 0x04 = MQTT 3.1.1
  连接标志: 0xC2 = 11000010
    - User Name Flag: 1
    - Password Flag: 1
    - Will Retain: 0
    - Will QoS: 0
    - Will Flag: 0
    - Clean Session: 1
  保活时间: 0x003C = 60秒

有效载荷:
  客户端ID长度: 0x000C = 12
  客户端ID: "MQTT_Client"
```

### PUBLISH 消息 (QoS 0)

```
输入: 30 11 00 0A 73 65 6E 73 6F 72 2F 74 65 6D 70 48 65 6C 6C 6F

解析:
固定头部:
  类型: 0x3 = PUBLISH
  DUP: 0
  QoS: 0
  RETAIN: 0
  剩余长度: 0x11 = 17字节

可变头部:
  主题长度: 0x000A = 10
  主题: "sensor/temp"

有效载荷:
  消息体: "Hello" (5 bytes)
```

### SUBSCRIBE 请求

```
输入: 82 0A 00 01 00 03 73 2F 74 00

解析:
固定头部:
  类型: 0x8 = SUBSCRIBE
  剩余长度: 0x0A = 10字节

可变头部:
  报文标识: 0x0001 = 1

有效载荷:
  主题过滤器长度: 0x0003 = 3
  主题过滤器: "s/t"
  QoS: 0x00 = 0
```

### 心跳报文

```
PINGREQ:  C0 00
  类型: 0xC = 12 (PINGREQ)
  剩余长度: 0

PINGRESP: D0 00
  类型: 0xD = 13 (PINGRESP)
  剩余长度: 0
```

## 常见问题诊断

### 连接问题

| 问题 | 可能原因 | 诊断方法 |
|------|----------|----------|
| 连接被拒绝 | ClientID冲突 | 检查Clean Session标志 |
| 认证失败 | 用户名/密码错误 | 验证连接标志和凭据 |
| 协议版本不匹配 | 版本号错误 | 检查协议级别字段 |

### 发布问题

| 问题 | 可能原因 | 诊断方法 |
|------|----------|----------|
| 消息丢失 | QoS 0 | 检查QoS级别设置 |
| 重复消息 | QoS 1重传 | 检查PUBACK是否返回 |
| 消息阻塞 | QoS 2未完成 | 检查PUBREC/PUBCOMP流程 |

### 订阅问题

| 问题 | 可能原因 | 诊断方法 |
|------|----------|----------|
| 收不到消息 | 主题不匹配 | 验证主题过滤器格式 |
| 权限不足 | ACL限制 | 检查Broker配置 |

## 端口与安全

| 端口 | 协议 | 说明 |
|------|------|------|
| 1883 | MQTT | 标准非加密端口 |
| 8883 | MQTTS | TLS加密端口 |
| 8080/8443 | WebSocket | MQTT over WebSocket |

## MQTT 5.0 新特性

1. **原因码**: 连接失败时返回具体原因
2. **会话/消息过期**: 支持过期时间
3. **主题别名**: 减少主题名传输
4. **用户属性**: 键值对元数据
5. **共享订阅**: 负载均衡
6. **请求/响应**: 内置响应主题

### 常见原因码

| 码 | 名称 | 说明 |
|----|------|------|
| 0x00 | Success | 成功 |
| 0x01 | Granted QoS 1 | 订阅成功(QoS 1) |
| 0x02 | Granted QoS 2 | 订阅成功(QoS 2) |
| 0x04 | Disconnect with Will Message | 正常断开 |
| 0x05 | No matching subscribers | 无订阅者 |
| 0x80 | Unspecified error | 未指定错误 |
| 0x81 | Malformed Packet | 格式错误 |
| 0x82 | Protocol Error | 协议错误 |
| 0x85 | Client Identifier not valid | ClientID无效 |
| 0x86 | Bad User Name or Password | 认证失败 |
