# IEC 61850 协议参考

## 协议概述

IEC 61850 是变电站自动化标准，包含多个子协议：

| 协议 | 用途 | 端口/标识 |
|------|------|----------|
| MMS | 客户端-服务器通信 | TCP 102 |
| GOOSE | 快速跳闸信号 | 以太网类型 0x88B8 |
| SV | 采样值传输 | 以太网类型 0x88BA |

## MMS 协议

### 协议栈

```
┌─────────────────────────────────────┐
│           IEC 61850 ACSI            │
├─────────────────────────────────────┤
│           MMS                       │
├─────────────────────────────────────┤
│           ISO Presentation          │
├─────────────────────────────────────┤
│           ISO Session               │
├─────────────────────────────────────┤
│           TPKT (RFC 1006)           │
├─────────────────────────────────────┤
│           TCP                       │
└─────────────────────────────────────┘
```

### TPKT 头部

```
┌──────────┬──────────┬─────────────────────┐
│  版本    │  保留    │       长度          │
│  1 byte  │  1 byte  │      2 bytes        │
│   0x03   │   0x00   │   (含头部)          │
└──────────┴──────────┴─────────────────────┘
```

### COTP 连接请求 (Connection Request)

```
输入: 03 00 00 16 11 E0 00 00 00 01 00 C0 01 0C C1 05 01 00 C2 06 01 00

解析:
TPKT:
- 版本: 0x03
- 长度: 22

COTP:
- 长度: 17
- PDU类型: 0xE0 (Connection Request)
- 目标引用: 0x0000
- 源引用: 0x0001
- 类选项: 0x00

参数:
- 0xC1: TSAP目标 (长度5)
- 0xC2: TSAP源 (长度6)
```

### MMS 服务

| 服务 | 说明 |
|------|------|
| Identify | 服务器标识 |
| GetServerDirectory | 获取逻辑设备列表 |
| GetDeviceDirectory | 获取逻辑节点列表 |
| GetDataDirectory | 获取数据对象列表 |
| GetDataValues | 读取数据值 |
| SetDataValues | 写入数据值 |
| GetDataSetDirectory | 获取数据集列表 |
| GetDataSetValues | 读取数据集值 |
| Report | 报告服务 |
| Control | 控制服务 |

## GOOSE 协议

### 以太网帧结构

```
┌──────────────────┬──────────────────┬──────────────┬─────────────┐
│   目标MAC        │    源MAC         │  以太网类型  │   APPID     │
│   6 bytes        │    6 bytes       │  0x88B8      │   2 bytes   │
├──────────────────┴──────────────────┴──────────────┴─────────────┤
│                        GOOSE PDU                                  │
├───────────────────────────────────────────────────────────────────┤
│   Length     │  Reserved1   │  Reserved2   │     GOOSE内容       │
│   2 bytes    │   2 bytes    │   2 bytes    │     可变长度        │
└───────────────────────────────────────────────────────────────────┘
```

### GOOSE PDU 结构 (ASN.1 编码)

```
GOOSE PDU:
├── gocbRef: GoCB引用名称
├── timeAllowedToLive: 存活时间(ms)
├── datSet: 数据集引用
├── goID: GOOSE标识
├── t: 时间戳
├── stNum: 状态号
├── sqNum: 序列号
├── test: 测试标志
├── confRev: 配置版本
├── ndsCom: 需要重新配置标志
├── numDatSetEntries: 数据条目数
└── allData: 数据值列表
```

### GOOSE 关键字段

| 字段 | 说明 |
|------|------|
| stNum | 状态号，数据变化时递增 |
| sqNum | 序列号，同一状态重发时递增 |
| timeAllowedToLive | 接收方等待时间，超时应告警 |
| test | 测试模式，正常应为FALSE |

### GOOSE 状态变化检测

```python
def check_goose_change(old_msg, new_msg):
    """检测GOOSE状态变化"""
    if new_msg['stNum'] != old_msg['stNum']:
        return "状态变化"
    elif new_msg['sqNum'] != old_msg['sqNum'] + 1:
        return "序列号不连续，可能丢包"
    else:
        return "心跳帧"
```

## SV (采样值) 协议

### 以太网帧结构

```
┌──────────────────┬──────────────────┬──────────────┬─────────────┐
│   目标MAC        │    源MAC         │  以太网类型  │   APPID     │
│   6 bytes        │    6 bytes       │  0x88BA      │   2 bytes   │
├──────────────────┴──────────────────┴──────────────┴─────────────┤
│                        SV PDU                                     │
├───────────────────────────────────────────────────────────────────┤
│   Length     │  Reserved1   │  Reserved2   │     SV内容          │
│   2 bytes    │   2 bytes    │   2 bytes    │     可变长度        │
└───────────────────────────────────────────────────────────────────┘
```

### SV PDU 结构

```
SV PDU:
├── noASDU: ASDU数量
├── smpCnt: 采样计数器
├── confRev: 配置版本
├── smpSynch: 同步标志
├── smpRate: 采样率
└── Sequence of ASDU:
    └── ASDU:
        ├── svID: SV标识
        ├── datSet: 数据集
        ├── smpMod: 采样模式
        ├── smpRate: 采样率
        └── Sequence of Data:
            └── 采样值 (通常为电流、电压)
```

## 解析示例

### MMS 读取请求

```
输入: 03 00 00 3F 02 F0 80 01 00 00 01 00 02 00 ...

解析:
TPKT:
- 版本: 0x03
- 长度: 63

ISO Session:
- 类型: 0x02 (DATA)
- 参数: 0xF0, 0x80

MMS:
- 服务类型: GetDataValues
- 域名: 指定的逻辑设备
- 变量名: 指定的数据对象
```

### GOOSE 报文

```
输入: 01 0C CD 01 00 00 00 00 00 01 88 B8 00 01 ...

解析:
以太网头:
- 目标MAC: 01:0C:CD:01:00:00 (组播地址)
- 源MAC: 00:00:00:00:00:01
- 以太网类型: 0x88B8 (GOOSE)

GOOSE头:
- APPID: 0x0001
- 长度: ...
- 保留: 0x0000 0x0000

GOOSE PDU (BER编码):
- gocbRef: "LD0/LLN0$GO$gocb1"
- stNum: 1
- sqNum: 1
- ...
```

## 常见问题

1. **连接超时**: 检查TCP 102端口、TSAP配置
2. **GOOSE丢包**: 检查网络延迟、交换机配置
3. **SV不同步**: 检查PTP时钟同步、采样率配置
4. **MMS服务拒绝**: 检查访问控制、数据模型配置
5. **报告不触发**: 检查触发条件、数据集配置

## 数据模型

IEC 61850 采用层次化数据模型：

```
IED (物理设备)
└── Logical Device (逻辑设备)
    └── Logical Node (逻辑节点)
        └── Data Object (数据对象)
            └── Data Attribute (数据属性)
```

### 常见逻辑节点

| 代码 | 名称 | 说明 |
|------|------|------|
| LLN0 | 逻辑节点零 | 设备级功能 |
| LPHD | 物理设备 | 物理设备信息 |
| XCBR | 断路器 | 断路器控制 |
| XSWI | 隔离开关 | 开关控制 |
| MMXU | 测量单元 | 测量值 |
| PTOC | 过流保护 | 过流保护功能 |
| PDIS | 距离保护 | 距离保护功能 |
