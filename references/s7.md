# S7 协议参考

## 协议概述

西门子 S7 协议用于 PLC 与上位机通信，基于 ISO-on-TCP (RFC 1006)。

### 协议栈

```
┌─────────────────────────────────────┐
│           S7 Protocol               │
├─────────────────────────────────────┤
│           COTP (ISO 8073)           │
├─────────────────────────────────────┤
│           TPKT (RFC 1006)           │
├─────────────────────────────────────┤
│           TCP (Port 102)            │
└─────────────────────────────────────┘
```

## 帧格式

### TPKT 头部

```
┌──────────┬──────────┬─────────────────────┐
│  版本    │  保留    │       长度          │
│  1 byte  │  1 byte  │      2 bytes        │
│   0x03   │   0x00   │   (含头部)          │
└──────────┴──────────┴─────────────────────┘
```

### COTP 头部

```
┌──────────┬──────────┬─────────────────────┐
│  长度    │ PDU类型  │      参数           │
│  1 byte  │  1 byte  │    可变长度         │
└──────────┴──────────┴─────────────────────┘
```

### S7 PDU 头部

```
┌──────────┬──────────┬──────────┬─────────────────────┐
│ Protocol │ ROSCTR   │  Reserved│     PDU Ref         │
│  1 byte  │  1 byte  │  2 bytes │      2 bytes        │
├──────────┴──────────┴──────────┴─────────────────────┤
│  Param Length │ Data Length  │      Error           │
│   2 bytes     │   2 bytes    │      2 bytes         │
└──────────────────────────────────────────────────────┘
```

### ROSCTR 类型

| 类型 | 值 | 说明 |
|------|-----|------|
| JOB | 0x01 | 作业请求 |
| ACK | 0x02 | 确认响应 |
| ACK_DATA | 0x03 | 数据响应 |
| USER_DATA | 0x07 | 用户数据 |

## 连接建立

### 1. COTP 连接请求

```
输入: 03 00 00 16 11 E0 00 00 00 01 00 C1 02 01 00 C2 02 01 02

解析:
TPKT:
- 版本: 0x03
- 长度: 22

COTP:
- 长度: 17
- PDU类型: 0xE0 (Connection Request)
- 目标引用: 0x0000
- 源引用: 0x0001
- 类选项: 0x00

参数:
- 0xC1 02 01 00: TSAP目标 (rack=0, slot=1)
- 0xC2 02 01 02: TSAP源
```

### 2. COTP 连接确认

```
输入: 03 00 00 16 11 D0 00 00 00 01 00 C1 02 01 00 C2 02 01 02

解析:
- PDU类型: 0xD0 (Connection Confirm)
```

### 3. S7 通信设置请求

```
输入: 03 00 00 19 02 F0 80 32 01 00 00 00 01 00 08 00 00 F0 00 00 01 00 01 03 C0

解析:
S7 PDU:
- Protocol: 0x32
- ROSCTR: 0x01 (JOB)
- Reserved: 0x0000
- PDU Ref: 0x0001
- Param Length: 8
- Data Length: 0

参数:
- 功能: Setup Communication
- PDU Size: 240
```

## 读写操作

### 读请求

```
输入: 03 00 00 1F 02 F0 80 32 01 00 00 00 05 00 0E 00 00 04 01 12 0A 10 02 00 30 00 01 84 00 00 20

解析:
S7 PDU:
- Protocol: 0x32
- ROSCTR: 0x01 (JOB)
- PDU Ref: 0x0005
- Param Length: 14
- Data Length: 0

参数:
- 功能码: 0x04 (Read)
- Item Count: 1
- Item 1:
  - Area: 0x84 (DB)
  - DB Number: 1
  - Type: 0x02 (Byte)
  - Count: 1
  - Address: 32
```

### 读响应

```
输入: 03 00 00 1C 02 F0 80 32 03 00 00 00 05 00 02 00 05 00 00 04 01 FF 09 00 08 00 00 AA

解析:
S7 PDU:
- Protocol: 0x32
- ROSCTR: 0x03 (ACK_DATA)
- PDU Ref: 0x0005
- Error: 0x0000 (无错误)

数据:
- Item Count: 1
- Item 1:
  - Error: 0x00 (成功)
  - Type: 0x09 (Byte)
  - Count: 1
  - Data: 0xAA
```

### 写请求

```
输入: 03 00 00 23 02 F0 80 32 01 00 00 00 06 00 0E 00 05 05 01 12 0A 10 02 00 30 00 01 84 00 00 20 00 03 FF 00 BB

解析:
参数:
- 功能码: 0x05 (Write)
- Item Count: 1
- Item 1:
  - Area: 0x84 (DB)
  - DB Number: 1
  - Type: 0x02 (Byte)
  - Count: 1
  - Address: 32

数据:
- Type: 0x03
- Count: 1
- Data: 0xBB
```

## 错误代码

### PDU 错误

| 代码 | 说明 |
|------|------|
| 0x0000 | 无错误 |
| 0x8100 | 不支持的协议版本 |
| 0x8200 | 不支持的消息类型 |
| 0x8300 | 不支持的参数 |

### Item 错误

| 代码 | 说明 |
|------|------|
| 0x00 | 无错误 |
| 0x01 | 硬件错误 |
| 0x03 | 对象不存在 |
| 0x05 | 地址无效 |
| 0x06 | 类型不支持 |
| 0x07 | 数据长度不匹配 |
| 0x0A | 对象不可访问 |

## 解析示例

### 读取 DB1.DBW0

```
请求报文:
03 00 00 21 02 F0 80 32 01 00 00 00 01 00 0E 00 00 04 01 12 0A 10 04 00 08 00 01 84 00 00 00

解析:
目标: DB1.DBW0 (DB1, Word, 地址0)
- Area: 0x84 (DB)
- DB Number: 1
- Type: 0x04 (Word)
- Count: 1
- Address: 0 (字节地址)
```

### 写入 M10.0 = TRUE

```
请求报文:
03 00 00 1F 02 F0 80 32 01 00 00 00 02 00 0E 00 03 05 01 12 0A 10 01 00 01 00 01 83 00 00 50 00 03 FF 00 01

解析:
目标: M10.0 (Merker, Bit, 地址10.0)
- Area: 0x83 (M)
- Type: 0x01 (Bit)
- Address: 10*8+0 = 80 (位地址)
- Data: 0x01 (TRUE)
```

## 常见问题

1. **连接拒绝**: 检查TSAP配置（机架/槽号）、PLC运行状态
2. **读写失败**: 检查地址格式、DB块是否存在、访问权限
3. **超时**: 检查网络连接、PDU大小配置
4. **数据类型错误**: 注意大小端，S7为大端格式
5. **地址计算错误**: 位地址 = 字节地址*8 + 位号

## 地址计算公式

```
字节地址 = 起始字节
位地址 = 字节地址 * 8 + 位号 (0-7)

读取 DB1.DBX10.5:
- Area: 0x84 (DB)
- DB Number: 1
- Type: 0x01 (Bit)
- Address: 10 * 8 + 5 = 85
```
