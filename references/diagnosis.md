## 1. 报文解析检查

### 基本检查项

| 检查项 | 说明 | 常见问题 |
|--------|------|----------|
| 帧起始 | 正确的起始字节 | 帧同步丢失 |
| 帧长度 | 实际长度与声明长度一致 | 截断、粘包 |
| 校验码 | CRC/校验和正确 | 传输错误 |
| 字段范围 | 字段值在合法范围内 | 配置错误 |

### 协议特定检查

#### Modbus
- 功能码有效性
- 寄存器地址范围
- CRC校验
- 响应数据长度与请求匹配

#### IEC 104
- 启动字节 (0x68)
- 序列号连续性
- 控制域格式正确性
- ASDU类型标识有效性

#### S7
- PDU完整性
- 内存区域有效性
- 数据类型匹配
- 错误码检查

## 2. 通信分析

### 请求-响应匹配

```
分析步骤:
1. 提取所有请求和响应
2. 根据事务ID/序列号配对
3. 计算响应时间
4. 检查是否有丢失或重复
```

### 响应时间统计

```python
def analyze_response_times(pairs):
    """分析响应时间统计"""
    times = [p.response_time - p.request_time for p in pairs]
    return {
        'min': min(times),
        'max': max(times),
        'avg': sum(times) / len(times),
        'timeout_count': sum(1 for t in times if t > TIMEOUT_THRESHOLD)
    }
```

### 通信质量指标

| 指标 | 计算方法 | 正常范围 |
|------|----------|----------|
| 成功率 | 成功响应数 / 总请求数 | > 99% |
| 平均响应时间 | 总响应时间 / 成功数 | < 设定阈值 |
| 超时率 | 超时数 / 总请求数 | < 0.1% |
| 重传率 | 重传数 / 总发送数 | < 0.5% |

## 3. 异常检测

### 常见异常模式

#### 3.1 无响应

```
现象: 发送请求后无任何响应

可能原因:
- 设备离线或断电
- 网络不通
- 地址错误
- 波特率/校验位不匹配
- 设备故障

排查步骤:
1. 检查物理连接
2. Ping测试网络连通性
3. 确认设备地址配置
4. 检查串口参数
5. 尝试其他命令
```

#### 3.2 超时

```
现象: 请求发送后超时

可能原因:
- 设备处理慢
- 网络延迟高
- 超时设置过短
- 设备负载高

排查步骤:
1. 增加超时时间
2. 检查网络延迟
3. 查看设备CPU/内存
4. 减少请求频率
```

#### 3.3 异常响应

```
现象: 设备返回异常/错误码

Modbus异常码:
- 01: 非法功能码
- 02: 非法数据地址
- 03: 非法数据值
- 04: 从站设备故障

IEC 104传输原因:
- 44: 未知的类型标识
- 45: 未知的传输原因
- 46: 未知的应用服务数据单元公共地址

排查步骤:
1. 检查功能码/类型标识是否支持
2. 检查地址范围
3. 检查数据格式和范围
4. 查阅设备文档
```

#### 3.4 数据错误

```
现象: 返回数据不符合预期

可能原因:
- 字节序问题(大小端)
- 数据类型解析错误
- 地址偏移错误
- 数据格式不匹配

排查步骤:
1. 确认字节序
2. 验证数据类型
3. 检查地址计算
4. 对比原始数据
```

#### 3.5 序列号异常 (IEC 104)

```
现象: 序列号不连续或重复

可能原因:
- 网络丢包
- 设备重启
- 并发问题
- 缓冲区溢出

排查步骤:
1. 检查网络质量
2. 检查设备日志
3. 检查流量控制
4. 调整窗口大小
```

#### 3.6 校验错误

```
现象: CRC/校验和验证失败

可能原因:
- 传输错误
- 线路干扰
- 波特率不匹配
- 数据截断

排查步骤:
1. 重新计算校验值
2. 检查线路质量
3. 确认波特率
4. 检查接地和屏蔽
```

## 4. 问题定位方法

### 分层排查法

```
┌─────────────────────────────────────┐
│          应用层 (数据内容)           │
│          问题的最终表现              │
├─────────────────────────────────────┤
│          协议层 (报文格式)           │
│          检查报文结构和字段          │
├─────────────────────────────────────┤
│          传输层 (网络/串口)          │
│          检查连通性和延迟            │
├─────────────────────────────────────┤
│          物理层 (硬件连接)           │
│          检查线缆、电源、端口        │
└─────────────────────────────────────┘
```

### 对比分析法

```
1. 获取正常通信的报文样本
2. 对比正常与异常报文
3. 找出差异点
4. 分析差异原因
```

### 二分法

```
1. 在通信链路中间点抓包
2. 判断问题在哪个方向
3. 缩小范围继续排查
4. 定位到具体设备或环节
```

## 6. 协议特定诊断

### Modbus 诊断

```python
def diagnose_modbus(request, response):
    """Modbus诊断"""
    issues = []

    # 检查功能码匹配
    if response.function_code != request.function_code:
        if response.function_code & 0x80:
            issues.append(f"异常响应: 错误码 {response.exception_code}")

    # 检查CRC
    if not verify_crc(response.raw):
        issues.append("CRC校验失败")

    # 检查数据长度
    expected_len = calculate_expected_length(request)
    if len(response.data) != expected_len:
        issues.append(f"数据长度不匹配: 期望{expected_len}, 实际{len(response.data)}")

    return issues
```

### IEC 104 诊断

```python
def diagnose_iec104(frames):
    """IEC 104诊断"""
    issues = []

    # 检查序列号连续性
    last_send_seq = -1
    for frame in frames:
        if frame.type == 'I':
            if last_send_seq >= 0:
                expected = (last_send_seq + 1) & 0x7FFF
                if frame.send_seq != expected:
                    issues.append(f"序列号不连续: 期望{expected}, 实际{frame.send_seq}")
            last_send_seq = frame.send_seq

    # 检查TESTFR
    if not has_recent_testfr(frames):
        issues.append("长时间未收到TESTFR，链路可能断开")

    # 检查传输原因
    for frame in frames:
        if hasattr(frame, 'cot') and frame.cot >= 44:
            issues.append(f"异常传输原因: COT={frame.cot}")

    return issues
```

### S7 诊断

```python
def diagnose_s7(response):
    """S7诊断"""
    issues = []

    # 检查PDU错误码
    if response.error != 0:
        issues.append(f"PDU错误: 0x{response.error:04X}")

    # 检查Item错误
    for i, item in enumerate(response.items):
        if item.error != 0:
            error_desc = {
                0x01: "硬件错误",
                0x03: "对象不存在",
                0x05: "地址无效",
                0x06: "类型不支持",
                0x0A: "对象不可访问"
            }.get(item.error, f"未知错误 0x{item.error:02X}")
            issues.append(f"Item {i}: {error_desc}")

    return issues
```

## 7. 常见故障速查表

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 完全无响应 | 物理连接问题 | 检查线缆、电源 |
| 偶发超时 | 网络波动 | 检查网络质量，增加超时 |
| 随机错误码 | 电磁干扰 | 加屏蔽、接地 |
| 数据偏移 | 地址计算错误 | 检查地址公式 |
| 字节序错误 | 大小端问题 | 确认字节序设置 |
| CRC错误 | 传输错误 | 检查线路质量 |
| 序列号异常 | 丢包/重传 | 检查网络、调整窗口 |
| 设备重启 | 电源/看门狗 | 检查电源稳定性 |
