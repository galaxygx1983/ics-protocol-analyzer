# IEC 60870-5-101 协议参考

## 帧格式 (FT1.2)

IEC 101 使用异步串行通信，帧格式有固定长度和可变长度两种。

### 固定长度帧

用于确认帧和请求链路状态：

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ 启动字节 │  控制域  │  地址域  │  校验和  │ 结束字节 │
│   10H    │   C      │   A      │   CS     │   16H    │
│  1 byte  │  1 byte  │  1 byte  │  1 byte  │  1 byte  │
└──────────┴──────────┴──────────┴──────────┴──────────┘
```

### 可变长度帧

用于传输数据：

```
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│ 启动字节 │  长度L   │ 启动字节 │  控制域  │  地址域  │  ASDU    │ 校验+结束 │
│   68H    │   L      │   68H    │   C      │   A      │ L-2 bytes│ CS + 16H │
│  1 byte  │  1 byte  │  1 byte  │  1 byte  │  1 byte  │          │  2 bytes │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
```

## 链路层控制域

### 主站→从站

```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│ PRM│ FCB│ FCV│ DFC│       功能码        │
│ 1  │ 1  │ 1  │ 1  │       4 bits        │
└────┴────┴────┴────┴────────────────────┘
```

| 字段 | 说明 |
|------|------|
| PRM | 1=主站发起，0=从站响应 |
| FCB | 帧计数位，交替0/1 |
| FCV | FCB有效标志 |
| DFC | 数据流控制 |
| 功能码 | 操作类型 |

### 从站→主站

```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│ PRM│ ACD│ DFC│        功能码            │
│ 1  │ 1  │ 1  │        5 bits           │
└────┴────┴────┴────────────────────────┘
```

| 字段 | 说明 |
|------|------|
| ACD | 访问请求，1=有一级数据 |
| DFC | 数据流控制，1=缓冲区满 |

### 功能码

| 功能码 | 主站→从站 | 从站→主站 |
|--------|----------|----------|
| 0 | 复位远方链路 | 确认 |
| 1 | 复位用户进程 | 链路忙 |
| 3 | 发送/确认用户数据 | - |
| 4 | 发送/无回答用户数据 | - |
| 8 | 访问请求 | - |
| 9 | 请求/响应请求链路状态 | 链路状态 |
| 10 | 请求/响应召唤一级数据 | 响应一级数据 |
| 11 | 请求/响应召唤二级数据 | 响应二级数据 |

## 地址域

- **链路地址**: 1字节或2字节（可配置）
- **公共地址**: ASDU中，1字节或2字节
- **信息对象地址**: 1/2/3字节（可配置）

## ASDU 格式

与 IEC 104 类似，但有以下区别：

1. **传输原因**: 1字节（IEC 104为2字节）
2. **公共地址**: 1字节或2字节（IEC 104固定2字节）
3. **信息对象地址**: 1/2/3字节（IEC 104固定3字节）

```
┌──────────────┬──────────────┬──────────────┬─────────────────────┐
│   类型标识   │  可变结构    │   传输原因    │      公共地址        │
│   (TypeID)   │   限定词     │    (COT)     │   (Common Addr)     │
│   1 byte     │   1 byte     │   1 byte     │   1 or 2 bytes      │
├──────────────┴──────────────┴──────────────┴─────────────────────┤
│                        信息体                                    │
│              (Information Objects)                               │
└──────────────────────────────────────────────────────────────────┘
```

## 类型标识

| TypeID | 名称 | 说明 |
|--------|------|------|
| 1 | M_SP_NA_1 | 单点信息 |
| 3 | M_DP_NA_1 | 双点信息 |
| 9 | M_ME_NA_1 | 测量值，标度化值 |
| 11 | M_ME_NB_1 | 测量值，浮点数 |
| 21 | M_ME_NA_1 | 不带品质归一化测量值 |
| 100 | C_IC_NA_1 | 总召唤命令 |
| 103 | C_CS_NA_1 | 时钟同步命令 |

## 解析示例

### 复位链路请求

```
输入: 10 40 01 41 16

解析:
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ 启动字节 │  控制域  │  地址域  │  校验和  │ 结束字节 │
│   10     │   40     │   01     │   41     │   16     │
└──────────┴──────────┴──────────┴──────────┴──────────┘

- 启动字节: 0x10 (固定长度帧)
- 控制域: 0x40
  - PRM=1 (主站发起)
  - FCB=0, FCV=0
  - 功能码=0 (复位远方链路)
- 地址域: 0x01 = 从站地址1
- 校验和: 0x41 (控制域+地址域)
- 结束字节: 0x16
```

### 复位链路确认

```
输入: 10 00 01 01 16

解析:
- 启动字节: 0x10 (固定长度帧)
- 控制域: 0x00
  - PRM=0 (从站响应)
  - ACD=0, DFC=0
  - 功能码=0 (确认)
- 地址域: 0x01
- 校验和: 0x01
```

### 请求一级数据

```
输入: 10 5A 01 5B 16

解析:
- 控制域: 0x5A
  - PRM=1 (主站发起)
  - FCB=1, FCV=1
  - 功能码=10 (请求一级数据)
```

### 响应一级数据（单点信息）

```
输入: 68 0E 68 08 01 01 01 01 03 00 01 00 01 0F 16

解析:
APCI:
- 启动字节: 0x68 (可变长度帧)
- 长度: 14
- 控制域: 0x08
  - PRM=0 (从站响应)
  - ACD=0, DFC=0
  - 功能码=8 (响应用户数据)

ASDU:
- TypeID: 0x01 = M_SP_NA_1
- VSQ: 0x01 = 1个信息对象
- COT: 0x03 = 突发
- 公共地址: 0x01
- 信息对象: SIQ = 0x01 (合)
```

## 校验和计算

```python
def checksum(data: bytes) -> int:
    """IEC 101 校验和计算"""
    return sum(data) & 0xFF
```

## 常见问题

1. **链路无法建立**: 检查波特率、校验位、地址配置
2. **FCB不匹配**: 主站FCB与上次不同，从站应响应
3. **DFC=1**: 从站缓冲区满，应暂停发送
4. **超时无响应**: 检查链路状态，重发复位帧
5. **校验和错误**: 重新计算，检查数据完整性

## 与 IEC 104 的区别

| 特性 | IEC 101 | IEC 104 |
|------|---------|---------|
| 传输层 | 串行通信 | TCP/IP |
| 端口 | 串口 | 2404 |
| 帧格式 | FT1.2 | APDU |
| 传输原因 | 1字节 | 2字节 |
| 公共地址 | 1/2字节 | 固定2字节 |
| 序列号 | 无 | 有 |
| 确认机制 | FCB | S帧/I帧 |
